# PartFlow — Учет и производство корпусных деталей

**PartFlow** — это современное, быстрое и полностью автономное веб-приложение для учета складских запасов и контроля этапов производства корпусных деталей.

Интерфейс спроектирован с учетом современных требований к дизайну: темная и светлая темы, эффекты матового стекла (glassmorphism), интерактивный дашборд со статистикой прогресса производства, плавной анимацией и полной адаптивностью под мобильные устройства.

---

## 🚀 Основные возможности

1. **Каталог деталей**: Добавление, редактирование и удаление деталей с указанием артикула/названия, материала (алюминий, сталь, оргстекло, пластик и др.), категории, плана производства и текстового описания (например, номера чертежа или особенностей покраски).
2. **Контроль производства и склада**:
   - Отдельный счетчик количества деталей **«В наличии»** (на складе).
   - Отдельный счетчик количества деталей **«Сделано»** (общее выполненное производство).
   - Быстрая корректировка значений кнопками `+` и `-` прямо на карточке детали.
3. **Автоматический расчет прогресса**: Визуальные индикаторы выполнения плана по каждой отдельной детали и общий анимированный круговой прогресс-бар по всему заказу.
4. **Фильтрация и умный поиск**:
   - Живой поиск по названиям и описаниям деталей.
   - Фильтрация по типам материалов и статусу готовности (Не начато, В процессе, Готово).
   - Сортировка по алфавиту, дате обновления, прогрессу и объему плана.
5. **История активности (Лог производства)**: Детальная лента событий с фиксацией времени каждого изменения (например: добавление новой детали, изменение остатков на складе, завершение производства).
6. **Резервное копирование (Экспорт/Импорт)**: Экспорт всей базы данных и лога событий в файл JSON для сохранения бэкапа и возможность быстро импортировать файл обратно для восстановления.
7. **Локальное хранение (LocalStorage)**: Все данные хранятся на вашем устройстве, не требуют интернета и не передаются на сторонние серверы.

---

## 🛠️ Технологический стек

- **HTML5**: Семантическая структура разметки.
- **Vanilla CSS**: Современные стили, кастомные переменные, CSS Grid, Flexbox, плавные переходы (`transitions`), анимации и отзывчивый дизайн без использования тяжелых фреймворков.
- **JavaScript (ES6+)**: Реактивное управление состоянием, интеграция с `localStorage`, валидация форм, парсинг файлов бэкапа и динамический рендеринг компонентов.

## 💻 Локальный запуск

Так как приложение является клиентским SPA (Single Page Application), оно не требует установки сторонних зависимостей.

### Вариант 1: Простое открытие файла
1. Скачайте или клонируйте проект.
2. Дважды кликните по файлу `index.html`, чтобы открыть его в любом современном браузере.

### Вариант 2: Запуск локального сервера (Рекомендуется для разработчиков)
Для корректной работы всех функций браузера (например, некоторых фич импорта файлов) рекомендуется запускать проект через локальный сервер:

**Используя Python (уже установлен в большинстве систем):**
```bash
python -m http.server 8000
```
После этого откройте в браузере: `http://localhost:8000`

**Используя Node.js (если установлен):**
```bash
npx serve .
```

## 📦 Развертывание на GitHub Pages

Вы можете бесплатно разместить это приложение на хостинге **GitHub Pages**, чтобы открывать его с телефона или планшета прямо в цеху:

1. Загрузите этот репозиторий в ваш аккаунт GitHub.
2. Перейдите в настройки репозитория (**Settings**) -> вкладка **Pages** (в левом меню).
3. В разделе **Build and deployment** установите источник: `Deploy from a branch`.
4. Выберите ветку `main` (или `master`) и папку `/ (root)`, затем нажмите **Save**.
5. Через 1-2 минуты GitHub предоставит вам публичную ссылку вида `https://<ваш-логин>.github.io/<имя-репозитория>/`.

## 💾 Структура данных в LocalStorage

Данные сохраняются под ключом `partflow-data` в следующем формате JSON:

```json
{
  "parts": [
    {
      "id": "part-1718012345678",
      "name": "Передняя панель корпуса",
      "material": "Алюминий",
      "category": "Панели",
      "target": 10,
      "stock": 4,
      "done": 7,
      "description": "Чертеж №12, порошковая покраска.",
      "createdAt": "2026-06-10T12:00:00.000Z",
      "updatedAt": "2026-06-10T12:15:00.000Z"
    }
  ],
  "logs": [
    {
      "id": "log-1718012399999",
      "partId": "part-1718012345678",
      "partName": "Передняя панель корпуса",
      "type": "info",
      "text": "Произведено деталей Передняя панель корпуса: +1 (Всего готово: 7 из 10)",
      "timestamp": "2026-06-10T12:15:00.000Z"
    }
  ]
}
```
 Настройки темы оформления сохраняются отдельно под ключом `partflow-theme` со значением `"dark"` или `"light"`.
